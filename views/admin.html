Mål: Ansatte logger seg inn med byggkon.no-kontoen sin (samme som Office 365 / Teams) — uten å skrive passord. Dashbordet vet hvem som er innlogget, og kan vise personlig dashboard automatisk.

📋 Sjekkliste for IT-ansvarlig
  1. Registrer app i Azure-portalen
  2. Hent Client ID, Tenant ID og Client Secret
  3. Legg til Microsoft Graph-tilganger (User.Read m.fl.)
  4. Legg inn 4 env-variabler på Railway
  5. Test innlogging

Tar ca. 15 minutter når man har admin-tilgang i Azure.

1️⃣ Registrer dashbordet i Azure
  1. Logg inn i portal.azure.com som global admin for byggkon.no
  2. Gå til Microsoft Entra ID (eller «Azure Active Directory»)
  3. Velg App registrations → + New registration
  4. Fyll inn:
    • Name: Bygg-Kon Dashboard
    • Supported account types: Accounts in this organizational directory only (byggkon.no only - Single tenant) ← VIKTIG. Dette låser innlogging til kun byggkon.no-kontoer.
    • Redirect URI: Plattform = Web, URL = https://<din-railway-url>/auth/microsoft/callback
  5. Klikk Register

Hva er Railway-URL-en? Det er URL-en dashbordet ditt kjører på i Railway, f.eks. https://byggkon-dashboard.up.railway.app. Du finner den øverst på Railway-prosjektsiden. Endre dette tilbake til <Application URL>/auth/microsoft/callback hvis dashbordet flytter.

2️⃣ Kopier ID-ene fra «Overview»

Etter at app-en er opprettet, åpne den. Under Overview finner du to verdier som dashbordet trenger:

  • Application (client) ID → går i env-variabelen MICROSOFT_CLIENT_ID
  • Directory (tenant) ID → går i env-variabelen MICROSOFT_TENANT_ID

Disse er ikke hemmelige — kan limes inn hvor som helst.

3️⃣ Lag en client secret
  1. I app-en, gå til Certificates & secrets
  2. Under Client secrets → klikk + New client secret
  3. Description: Bygg-Kon dashboard
  4. Expires: 24 months (eller velg «Custom» for lengre)
  5. Klikk Add
  6. VIKTIG: Kopier Value-feltet straks — Azure viser det kun én gang.
    Dette går i env-variabelen MICROSOFT_CLIENT_SECRET.

⚠️ Hvis du mister verdien må du lage en ny secret. Du kan ikke hente den frem igjen.

4️⃣ Sett opp API-tilganger
  1. I app-en, gå til API permissions
  2. De fleste av disse skal være forhåndsvalgt — om ikke: klikk + Add a permission → Microsoft Graph → Delegated permissions og legg til:
    • openid
    • email
    • profile
    • User.Read
  3. Klikk Grant admin consent for byggkon.no (krever global admin) — det er en blå knapp etter at permissions er lagt inn

User.Read gir oss bare lov til å lese brukerens eget profilbilde, navn og e-post — ingenting annet.

5️⃣ Legg inn 4 env-variabler på Railway
  1. Logg inn på Railway.app
  2. Velg byggkon-dashboard-tjenesten
  3. Åpne Variables-fanen
  4. Klikk + New Variable for hver av disse:
VariabelVerdi
MICROSOFT_CLIENT_IDApplication (client) ID fra steg 2
MICROSOFT_TENANT_IDDirectory (tenant) ID fra steg 2
MICROSOFT_CLIENT_SECRETValue fra client secret (steg 3)
MICROSOFT_REDIRECT_URIDin Railway-URL + /auth/microsoft/callback
OAUTH_ALLOWED_DOMAIN (valgfri)Standard er byggkon.no — endre hvis det skal være annet

Railway redeployer automatisk når variabler endres. Vent 30–60 sekunder etter siste lagring før du tester.

6️⃣ Test innloggingen
  1. Gå til dashbordets login-side (logg ut først hvis du er innlogget)
  2. Du skal nå se en «Logg inn med Microsoft»-knapp øverst på siden
  3. Klikk den. Du blir sendt til Microsoft. Hvis du allerede er innlogget i Office 365, blir du sendt rett tilbake. Ellers velger du byggkon.no-kontoen.
  4. Du skal lande på dashbordets forside som innlogget
  5. I toppmenyen vises navnet ditt (og at du er innlogget med Microsoft)

Hvis det ikke virker, sjekk vanlige feil:

  • «state-mismatch» → Cookies kan ha utløpt. Last login-siden på nytt og prøv.
  • «wrong-domain» → Brukeren prøvde med en konto som ikke er @byggkon.no
  • «token-exchange» → Client secret er feil, eller utløpt. Lag en ny.
  • «oauth-disabled» → Env-variabler ikke satt. Sjekk Railway → Variables.
  • Knappen vises ikke → Railway har ikke redeployet, eller en av de 4 variablene mangler.
🔐 Hva slags tilgang gir vi Azure-appen?

Appen får kun lov til å lese identiteten til den som logger inn (navn, e-post, jobbtittel og profilbilde). Den får IKKE tilgang til:

  • Brukerens e-post eller kalender
  • SharePoint-filer eller Teams-meldinger
  • Andre brukeres data
  • Endring av noe som helst i Entra ID

Det er ren autentisering — «hvem er du?» — ikke datatilgang. All data om jobben (timer, prosjekter, regnskap) hentes fortsatt fra Tripletex og lagres i Bygg-Kons egne systemer.
